# План и расписание — Java backend (2 месяца)

Цель: подготовиться к стажировке на Java backend — подтянуть теорию, довести 3–4 Java‑проекта до презентабельного состояния, улучшить английский до разговорного уровня B1+ для собеседований.

Общий принцип
- Фокус на качестве, не на количестве: 3 основных проекта (rgr, the-witcher, Christmas) + довести `LOL` как демо.
- Ежедневные короткие блоки: 2–3 учебных блока в день по 45–90 минут с перерывами.
- Чередование: тяжёлые теоретические дни ↔ лёгкие документирование/английский.
- Здоровье важнее: если плохо — 30–60 минут лёгкой работы (пример: правка README или грамматика).

Недели (сводно)
- Неделя 1: Core Java (OOP, коллекции, Stream API), подготовить `rgr` README и `.env.example`.
- Неделя 2: Concurrency и JVM (потоки, синхронизация, GC), добавить unit/integration тесты в `rgr`.
- Неделя 3: Spring Boot (DI, контроллеры, конфигурация), начать бэкенд для Christmas (CRUD).
- Неделя 4: Spring Data JPA, миграции, docker-compose для Christmas.
- Неделя 5: Безопасность (JWT/OAuth2/Keycloak обзор), в `rgr` улучшить auth-документацию.
- Неделя 6: Микросервисы (архитектура, логирование, resiliency), написать раздел архитектуры в `rgr`.
- Неделя 7: Системный дизайн / оптимизация (индексы, кеширование), подготовить ответы + диаграммы.
- Неделя 8: Мок‑интервью, финальная полировка проектов, запись демонстраций (gif/видео).

Еженедельный ритм (пример)
- Пн–Пт: 2 часа код/теория утром + 45 минут английский вечером.
- Сб: 2–3 часа проект/полировка + 1 час speaking practice (запись ответов).
- Вс: отдых или лёгкий review.

Ежедневный мини‑чеклист (30–90 мин)
- Техническая цель (например, прочитать главу/реализовать endpoint)
- 15–30 мин английский (Murphy или vocab + 5 мин проговаривание)
- 10–15 мин оформить заметку в Obsidian — конспект/фразы/README-строка

Файлы и артефакты к финалу
- README с quickstart и .env.example для `rgr` и Christmas
- 2–3 теста (unit/integration) для ключевых сервисов
- Runnable JAR или docker-compose для Java‑проектов
- Демозапись 1–2 минут и 5–7 фраз на английском о проекте


## Рекомендации
- Используйте таймер Pomodoro (45/15) для глубоких блоков
- В Obsidian ведите ежедневные краткие записи прогресса (3–5 bullet)
- Не забывайте про отдых и отслеживайте своё состояние
